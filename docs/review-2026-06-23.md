# review-2026-06-23.md — full QA / SEO / blindspot / UI-UX review

Honest punch-list from the 2026-06-23 review pass. `[ ]` = open, `[x]` = done since.
Priorities: **P1** = before a real (paying) launch · **P2** = soon after · **P3** = polish/nice-to-have.
Companion docs: [qa.md](qa.md) (checklist), [qa-audit.md](qa-audit.md) (evidence), [roadmap.md](roadmap.md), [plato-card.md](plato-card.md).

## Overall feel
Strong: cohesive, on-brand (ink/orange), pixel-matched to the mockups, fast after the perf work, and the
glass touches on `/card` + the diner action bar give a premium edge. Reads like a real product. The gaps
below are mostly **operational hardening, SEO polish, and Plato Card v1 loose ends** — not structural.

## P1 — before a real launch
- [ ] **Error monitoring (Sentry or similar).** Production errors are invisible — we hit a silent prod-only 500
  earlier (the impersonation bug) that only luck surfaced. Highest-leverage gap.
- [ ] **Real owner account, full flow.** 0 real tenant members exist (all admin impersonation). The invite →
  set-password → login → edit-menu path (Resend) is untested on a real inbox. Verify with one real restaurant.
- [ ] **Live 2-tenant RLS isolation test** (qa §1) with two real owners.
- [ ] **Ops:** confirm Supabase backups on; attorney sign-off on Terms/Privacy/agreement drafts.

## P2 — soon after
- [ ] **SEO:** `/card` and `/book` are not in `sitemap.ts`; `/card` has no OpenGraph (no rich social preview).
  Add both to the sitemap; give `/card` OG tags + an OG image. (quick)
- [ ] **Plato Card — host the strip image.** Logo is now self-hosted (`/brand/plato-pass-icon.png`), but the
  pass **strip is still a weserv proxy URL** → breaks if weserv is down when PassBuddy regenerates. Host a
  designed strip on platodigital.io + PATCH it.
- [ ] **Plato Card "Members" count** is a static placeholder (`15k/mo` + partner count). Wire a real add-count
  by tracking "Add to Apple Wallet" taps (cookieless event).
- [ ] **Real-iPhone diner test:** glass action-bar perf under scroll + autoplaying video; HEIC handling; the
  floating bottom bar not covering content on small screens.
- [ ] **Flagship with real video:** demo menu has `video_id = null` everywhere → Menu play-overlays + most-played
  run on seeded data. Film one real restaurant through Bunny end-to-end to validate the video pipeline.
- [ ] **Custom domains (Growth perk) still NOT built** — promised by the tier, not live (no host-routing middleware).

## P3 — polish / nice-to-have
- [ ] Plato Card loose ends: scheduled blasts stay `scheduled` in our record forever (no delivery webhook to flip
  to `sent`); promo invoices have no generated PDF (muted chip — `lib/invoice-pdf.ts` exists, not wired into
  approveBlast); cadence cap covers admin network blasts but not paid approvals; partner self-listing isn't
  admin-vetted (a restaurant can list itself + set any perk).
- [ ] Notification-budget readout is hardcoded, not the live PassBuddy quota (no quota API).
- [ ] Stripe / card payments not wired (invoice-only) — fine for now.
- [ ] Accessibility: accepted white-on-orange contrast (~2.9:1, below AA) by choice; formal AA needs darker CTAs.
- [ ] Owner/admin dashboards are EN-only (diner page is EN/ES).
- [ ] Mobile-check the new admin/owner Plato Card pages (3-col metric grid may cramp on a phone).
- [ ] DB index tune-up — deferred; revisit at scale (roadmap #4).

Tenant RLS isolation (incl. new `wallet_blasts`), the privileged-column trigger, service-role-never-on-client,
per-tenant menu `generateMetadata`/OpenGraph + schema.org Restaurant/Menu JSON-LD, the impersonation fix
(route handlers), glass legibility (dark labels on frosted bar / white text on dark gradient), lint + build clean.
